# Кета — установка на телефон (PWA через GitHub Pages)

Устанавливаемое веб-приложение: после установки работает офлайн и открывается с иконкой как обычное приложение.

## Что в папке
- `index.html` — само приложение (всё внутри одного файла)
- `manifest.webmanifest` — имя и иконки приложения
- `service-worker.js` — офлайн-кэш
- `icon-*.png` — иконки
- `.nojekyll` — отключает обработку Jekyll на GitHub Pages (**обязательно**, не удаляйте)

## Публикация через GitHub Pages

1. Создайте репозиторий на github.com (например `keta`).
2. Загрузите **содержимое папки `pwa`** в корень репозитория
   (файлы `index.html`, `manifest.webmanifest`, `service-worker.js`, иконки и `.nojekyll` — именно сами файлы, не вложенную папку).
3. В репозитории: **Settings → Pages**.
4. В разделе *Build and deployment* → Source выберите **Deploy from a branch**,
   ветка `main`, папка `/ (root)` → **Save**.
5. Через 1–2 минуты сайт будет доступен по адресу:
   `https://ВАШ_ЛОГИН.github.io/keta/`
6. Откройте этот адрес **на телефоне**.

## Добавить на экран «Домой»
- **iPhone (Safari):** «Поделиться» → «На экран «Домой»».
- **Android (Chrome):** меню ⋮ → «Установить приложение».

После установки приложение запускается на весь экран без адресной строки и работает без интернета. Записи хранятся прямо на устройстве.

## Что уже учтено для работы PWA
- Все пути относительные — работает в подпапке `username.github.io/keta/`.
- `manifest`: `start_url` и `scope` заданы относительно папки (`.`).
- Service worker регистрируется относительным путём → его scope охватывает приложение.
- Иконки 192/512 + maskable, `theme_color`, `display: standalone`.
- `apple-touch-icon` и мета-теги для добавления на экран «Домой» в iOS.
- `.nojekyll` — чтобы GitHub отдавал файлы как есть.
- HTTPS обеспечивает сам GitHub Pages (требование для service worker и установки).
